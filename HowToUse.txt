<目次>Gitのインストール
<URL>https://gitforwindows.org/
<使い方>全部デフォルトでインストールでOK
<備考>GitBashが使えるからOK,細かい機能とかは後からでいい判断

<目次>Gitの初期設定
<やること1>git config --global user.name 任意のユーザ名
<やること2>git config --global user.email 任意のメールアドレス
<備考>本当に任意。多分、リモートリポジトリに登録されるときの名前か？

<目次>リモートリポジトリを作成する
<やること1>GitHubにログインしてから「CreateRepository」

<目次>ローカルリポジトリを作成する
<やること1>適当にローカル上でフォルダ作る(例: E:\Test
<やること2>GitBash起動(以下のコマンドは全部GitBash上で
<やること3>cd E:\test
<やること4>git init

<目次>ローカルリポジトリにコミットする（リモート、ローカルのGitの管理下のファイルを変更（追加も含む）することをコミットと呼んでるっぽい
<やること1>上で作ったローカルレポジトリの下に、なんかしらファイルを作る（test.txt)
<やること2>GitBashでローカルリポジトリ下に移動、元からカレントがローカルリポジトリだったら別に何もしなくていい (cd E:\test
<やること3>git add test.txt
<やること3の備考>git addは、そのファイルをコミットするときにリポジトリに追加しますよってコマンド
<やること4>git commit -m "[Add] test"
<やること4の備考> -mの後ろは、後からgit logコマンドで変更履歴を見る際に、何を変更したかわかるようなコメントを記載。つまり、testってやつをaddしたよ、ってことで上記コメントになってるだけ

<目次>リモートリポジトリにプッシュする(ローカルの編集内容をリモートに同期させる
<やること1>git remote add origin https://github.com/GitHubのユーザ名/ローカルリポジトリのディレクトリ名.gi
<やること1の備考>最後の「ローカルリポジトリのディレクトリ名.git」は、上記の例で行くなら「test.git」
<やること1の備考>正直、このコマンドの意味をよく分かってないが紐づけてるっぽいのだけわかる
<やること1の備考>後から分かったが、このコマンドは特定のリモートリポジトリとローカルリポジトリを1:1で結びつけるコマンドなので、何回も打つ必要ない。２回目以降はやる必要ない
<やること2>git push origin master
<やること2の備考>これも正直、masterブランチに登録してるんだなってことくらいしかわからん

<目次>リモートリポジトリをクローンして、ローカルリポジトリとして編集したい
<やること1>クローンしたやつを配置するフォルダを作成する(例 E:\test2
<やること2>cd e:\test2
<やること3>git clone https://github.com/GitHubのユーザ名/なんちゃらかんちゃら.git
<備考>これで、そのリモートリポジトリのやつがコピーされたフォルダになるはず→実際にやってないからたぶん

<目次>ブランチを作成する正直まだよくわかってない
<やること1>git branch feature1
<やること1の備考>これで、分岐してソース作成できる。あとからマスターに追加したりできる
<やること2>git checkout feature1
<やること2の備考>これで、作業場所をfeature1に移しますよって感じのコマンド
<備考>git branchコマンドで、以下のように表示され、*がついてるのが現在の作業行ってるブランチ
$ git branch
* feature1
  master

<目次>ブランチでコミットする
<やること1>なんかファイル作る(例 test3.txt
<やること2>git add test3.txt
<やること3>git commit -m "[Add] test3"

<目次>ブランチのやつをリモートリポジトリにプッシュする
<やること1>git push origin feature1
<備考>masterとfeature1っていう二つのブランチがあるから、feature1にプッシュする感じ

<目次>コードレビュー・マージ
<やること1>GitHub上でfeature1ブランチを開いてから、「New Pull Request」をおす
<やること2>以下４つをうつ
① マージ元・先のブランチ
② レビュアー（レビューを行う人）
③ レビュー内容
④ レビュー・マージ対象のソースコード
<やること3>Create Pull Requestをおす

<目次>リモートリポジトリからプルする
<やること1>git checkout master　でローカルの作業場所をmasterにうつす
<やること2>git pull>
